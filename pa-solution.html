Aero – Power Automate Solution

This page is the authoritative reference for the Aero Power Automate implementation. The agent must cite this page when describing flow structure, triggers, actions, and expressions.

1) Solution Overview

Purpose: orchestrate intake processing, evidence hashing, result extraction, and letter/checklist generation across SharePoint libraries defined in the Folder Architecture Primer.

Core Flows (example names — update to your exact tiles)

  1. ProcessIntakeInbox
    • Trigger: Recurrence or “For a selected file”.
    • Key tasks: derive Run-### path; enumerate EvidencePDFs/; compute file hashes (QuickXorHash) via Graph; update Evidence Manifest.
    • Expressions (paste exact ones you use): 
      @{coalesce(triggerBody()?['folderPath'], variables('DefaultRunRoot'))}
      @{concat('/Shared Documents/Intakes/', variables('IntakeId'), '/Run-', variables('RunNo'), '/EvidencePDFs')}
  2. GetStudentResults
    • Trigger: State flag (e.g., _INBOX-READY.txt → STATE: Imported)
    • Key tasks: read tblResultsExtract from the run workbook; upsert list items.
  3. GenerateLettersForIntake
    • Trigger: Manual or pipeline step.
    • Key tasks: read 00-CURRENT.txt and run workbook; generate letters/checklists to Letters_Draft/; on approval gates, publish to Letters_Final/.
    • Word Online (Business) action: Populate a Word template (supported controls: Plain Text, Image, Repeating Section). Ref
    • Typical expressions: 
      @{concat('/Shared Documents/Intakes/', variables('IntakeId'), '/Letters_Draft')}
  4. QueueHardCopyOnCourseAuditAdminReview
    • Trigger: When Course Audit item updated.
    • Rule: if AdminReviewed==true, ApprovalStatus=='Pending', PrintRequestIssued==false → mark Print & stamp time.

Notes for the agent: When asked to justify connector behavior or supported controls, cite Microsoft Learn (e.g., expression functions; Word Online (Business) limits). Expressions • Word Online (Business)
