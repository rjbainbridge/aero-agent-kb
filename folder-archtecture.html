SharePoint Folder Architecture — Authoritative Map

Libraries & Roles

  • Documents — operational outputs, reviews, final letters, and run workspaces.
  • AeroEvidence — input‑only (evidence inputs, RunPublish inputs, logs). No generated outputs here.

Intake Root (Documents)

/Shared Documents/Intakes/<Intake ID>/
    Letters_Draft/
    Letters_Final/
    Run-###/
    00-CURRENT.txt
    _Inbox/
  • Letters_Draft/ — working versions for Admin → PRT → ISA review.
  • Letters_Final/ — approved/issued versions only (post‑gates).
  • 00-CURRENT.txt — active run metadata pointer written during publish.
  • _Inbox/ — optional operator staging.

Run‑Level Workspace (Documents)

/Shared Documents/Intakes/<Intake ID>/Run-###/
    EvidencePDFs/
    Exports/
    Workbook/
  • EvidencePDFs/ — standardized Attendance/Transcript PDFs (copied/renamed by macros when importing).
  • Exports/ — RunPublish-*.xlsx and other run exports.
  • Workbook/ — authoritative locked cohort workbook copy for the run.

AeroEvidence (Inputs Only)

/Accreditation Documents/Intakes/<Intake ID>/
    EvidencePDFs/
    Workbook/
    Logs/
    Exports/
    RunPublish.xlsx
  • Store input evidence and RunPublish inputs for audit; never store generated letters here.

Routing Rules (No Ambiguity)

  • Draft letter/checklist → /Shared Documents/Intakes/<Intake ID>/Letters_Draft/
  • Final letter/checklist → /Shared Documents/Intakes/<Intake ID>/Letters_Final/
  • Attendance/Transcript PDFs for a run → .../Run-###/EvidencePDFs/
  • Run exports & workbook → .../Run-###/Exports/ and .../Run-###/Workbook/
  • Run Publish inputs → /Accreditation Documents/Intakes/<Intake ID>/RunPublish.xlsx

Letters are generated outputs. They are stored directly under the Intake folder in Documents (Draft → Final), not under Run-###, and never in AeroEvidence.
